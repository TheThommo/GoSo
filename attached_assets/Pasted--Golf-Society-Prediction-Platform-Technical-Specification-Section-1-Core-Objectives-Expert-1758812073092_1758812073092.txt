# Golf Society Prediction Platform - Technical Specification

## Section 1: Core Objectives & Expert Recommendations

### Primary Goal
Build a comprehensive golf society prediction platform that enables real-world tournament management with betting-style predictions using a fictitious token system. The platform must be mobile-first, vibrant, and scalable for multiple concurrent tournaments.

### Technical Stack Recommendation

**Frontend: Next.js 14 with TypeScript**
- Reasoning: Built-in SSR/SSG for SEO and performance, excellent mobile responsiveness, TypeScript for type safety
- UI Framework: Tailwind CSS + shadcn/ui components for rapid development and consistent mobile-first design
- State Management: Zustand (lightweight, perfect for Replit's constraints)

**Backend: Next.js API Routes + tRPC**
- Reasoning: Unified codebase reduces complexity in Replit, tRPC provides end-to-end type safety
- Authentication: NextAuth.js with email/magic link providers
- Real-time: Socket.io for live updates and admin broadcasts

**Database: PostgreSQL via Neon**
- Reasoning: Excellent Replit integration, full ACID compliance, supports complex queries for analytics
- ORM: Prisma for type-safe database operations and seamless schema management

**Deployment: Replit + Vercel Integration**
- Static assets via Vercel's CDN for optimal mobile performance
- Environment variables managed through Replit Secrets

### Data Model Design

```sql
-- Core Users & Roles
Users (id, email, name, created_at, subscription_status, token_balance)
UserRoles (user_id, role, game_id?, created_at) -- master_admin, game_admin, player, spectator

-- Game Management
Games (id, name, course_name, location, tees, start_date, days_count, total_golfers, interval_minutes, is_locked, created_by, created_at)
TeeTimeSlots (id, game_id, slot_time, position_order)

-- Player Registry (Persistent across games)
PlayerRegistry (id, first_name, last_name, nickname, gender, base_handicap, email?, created_at)
GamePlayers (id, game_id, player_id, adjusted_handicap, tee_slot_id, tee_position)

-- Prediction System
PredictionScopes (id, game_id, scope_type, entity_ids[], odds, is_active) -- individual/pair/group
PlayerPredictions (id, user_id, game_id, first_place_scope_id, second_place_scope_id, third_place_scope_id)
TokenBets (id, user_id, game_id, scope_id, token_amount, potential_payout)

-- Results & Payouts
GameResults (id, game_id, final_rankings_json, payout_multipliers_json, finalized_at)
UserTransactions (id, user_id, game_id, transaction_type, amount, description, created_at)

-- Indexes for Performance
CREATE INDEX idx_games_start_date ON Games(start_date);
CREATE INDEX idx_user_roles_game ON UserRoles(game_id, role);
CREATE INDEX idx_predictions_user_game ON PlayerPredictions(user_id, game_id);
```

**Step 2: Player Management Interface**

- **Message Templates**: Pre-configured updates (hole-by-hole, weather alerts, results)
- **Push Notifications**: SMS/email integration for critical updates

### Scalability & Security Plan

**Performance Optimization**
- **Database Sharding**: Partition games by date/region for large-scale deployment
- **Caching Strategy**: Redis integration for frequently accessed data (leaderboards, odds)
- **CDN Implementation**: Static asset optimization for global mobile performance
- **Lazy Loading**: Progressive data loading for mobile bandwidth optimization

**Security Framework**
- **Data Encryption**: AES-256 for sensitive data, TLS 1.3 for all communications
- **Rate Limiting**: API throttling to prevent abuse, bot protection
- **Input Validation**: Comprehensive sanitization and validation at all endpoints
- **Audit Logging**: Complete user action tracking for compliance and debugging
- **GDPR Compliance**: Data export/deletion tools, consent management
